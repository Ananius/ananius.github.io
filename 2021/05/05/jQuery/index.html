1.2 jQuery中的顶级对象$

  • $是 jQuery 的别称，在代码中可以使用 jQuery 代替，但一般为了方便，通常都直接使用 $ 。
  • $是jQuery的顶级对象，相当于原生JavaScript中的 window。把元素利用$包装成jQuery对象，就可以调用jQuery 的方法。

1.3 jQuery 对象和 DOM 对象

​ 使用 jQuery 方法和原生JS获取的元素是不一样的，总结如下 :

  1. 用原生 JS 获取来的对象就是 DOM 对象
  2. jQuery 方法获取的元素就是 jQuery 对象。
  3. jQuery 对象本质是： 利用$对DOM 对象包装后产生的对象（伪数组形式存储）。

注意：

只有 jQuery 对象才能使用 jQuery 方法，DOM 对象则使用原生的 JavaScirpt 方法。

​ 停止动画排队的方法为：stop() ;

    • 该方法只能获取。

    • scrollTop() 方法设置或返回被选元素被卷去的头部。

    12.1 案例：toDoList 分析

    1. 刷新页面不会丢失数据，因此需要用到本地存储localStorage
    2. 核心思路：不管按下回车，还是点击复选框，都是把本地存储的数据加载到页面中，这样保证刷新关闭页面不会丢失数据
    3. 存储的数据格式（数组对象）：var todoList = [{title:’xxx’,done:false}]
    4. 注意点一：本地存储 localStorage 里面只能存储字符串格式，因此需要把对象转换为字符串 JSON.stringify(data)
    5. 注意点二：获取本地存储数据，需要把里面的字符串转换为数组对象格式 JSON.parse() 我们才能使用里面的数据

    12.2 步骤1：前期准备工作

    因为案例需要频繁的读取和存储数据到本地，所以需要编写两个方法

    1、读取本地存储数据：编写getData()方法，读取本地存储的JSON格式的字符串，返回一个数组，如果本地没有数据，则返回空数组

    2、保存到本地存储数据：编写saveData()方法，用于接收数组对象数据，将数据转成 JSON格式字符串，保存到本地

    12.4 步骤3：本地存储渲染加载到页面

    因为案例需要频繁的渲染本地数据，所以需要编写一个load()方法

    1. 先要读取本地存储数据。（调用之前的getData()方法）
    2. 遍历数组数据，数组中有几个数据，就创建几个 li ，添加到ul 和 ol 中
    3. 创建 li 之前先判断数组对象的done属性值，是true就存到ul中，是false就存到ol中
    4. 每次渲染之前，先把原先里面 ol 的内容清空，然后渲染加载最新的数据

    12.5 步骤4：删除数据

    1. 点击里面的a链接，不是删除的li，而是本地存储对应的数据
    2. 核心原理：先获取本地存储数据，删除对应的数据，保存给本地存储，重新渲染页面
    3. 我们可以给a链接添加id属性来获取索引号
    4. 根据这个索引号删除相关的数据—-数组的splice(index, 1)方法
    5. 存储修改后的数据，然后存储给本地存储
    6. 重新渲染加载数据列表
    7. 因为a是动态创建的，我们使用on方法事件委托绑定事件

    12.6 步骤5：正在进行和已完成选项切换

    1. 当我们点击了小的复选框，修改本地存储数据，再重新渲染数据列表。
    2. 点击之后，获取本地存储数据和索引号。
    3. 修改对应数据属性 done 为当前复选框的checked状态。
    4. 之后保存数据到本地存储
    5. 重新渲染加载数据列表

    12.7 步骤6：统计正在进行个数和已经完成个数

    1. 在我们load 函数里面操作
    2. 声明2个变量 ：todoCount 待办个数 doneCount 已完成个数
    3. 当进行遍历本地存储数据的时候， 如果 数据done为 false， 则 todoCount++, 否则 doneCount++
