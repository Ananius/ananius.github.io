2.13 分支语句

v-if 使用场景

  • 多个元素通过条件判断==显示==或者==隐藏==某个元素或者多个元素

v-show 和 v-if 的区别 ▲

  • 相同点：二者都可以动态控制 dom 元素的显示和隐藏

  • 不同点：

    • v-if 是动态向 dom 树中添加或删除 DOM 元素，控制元素是否渲染到页面上，v-show 是通过设置 DOM 元素的 display 属性控制显示隐藏，DOM 元素保留，已经将我们的元素渲染待页面中
    • v-if 适合运营条件不大可能改变，v-show 适合频繁切换
    • v-if 是惰性的，如果初始条件不成立，就什么都不会执行，只有在条件第一次变为真时才开始局部编译。v-show 在任何条件下都被编译，然后被缓存

  • v-show 本质就是标签 display 设置为 none，控制隐藏

    • v-show 只编译一次，后面其实就是控制 css，而 v-if 不停的销毁和创建，故 v-show 性能更好一点。

  • v-if 是动态的向 DOM 树内添加或者删除 DOM 元素

    • v-if 切换有一个局部编译/卸载的过程，切换过程中合适地销毁和重建内部的事件监听和子组件

4- bind 和 inserted 的区别 ▲

  • bind 是在 dom 树绘制前调用
  • 在元素刚绑定元素的时候,还没有插入到 dom 中去,这时候,调用 focus 方法没有作用,即 focus 放在 bind 中是不起作用的
  • bind 方法 绑定当前指令的元素， ==在内存中被创建的时候就会被立即调用；指令所绑定到的元素，还没有被插入父节点中==；
  • 一般把==样式相关==的设置都定义在 bind 方法中
  • inserted 在 dom 树绘制后调用
    • 绑定当前指令的元素，==被插入到 DOM 树之后才会被调用；当指令所绑定到的元素，被插入到文档中的父节点时候，会立即触发 inserted 方法==
    • 一般把==行为相关== 的设置都定义在 inserted 方法中

3.4 计算属性 computed

为什么需要计算属性？

  • 模板中放入太多的逻辑会让模板过重且难以维护 使用计算属性可以让模板更加的简洁
  • 计算属性是基于它们的响应式依赖进行缓存的
  • computed 比较适合对多个变量或者对象进行处理后返回一个结果值，也就是数多个变量中的某一个值发生了变化则我们监控的这个值也就会发生变化

基本用法：

计算属性和方法的区别

  • 计算属性是基于他们的依赖进行缓存的，方法不存在缓存
  • computed 是属性调用，而 methods 是函数调用

计算属性和方法的区别 ▲

  1. 计算属性本质上是包含 getter 和 setter 的方法。当获取计算属性时，实际上是在调用计算属性的 getter 方法，vue 会收集计算属性的依赖，并缓存计算属性的返回结果，只有当依赖变化后才会进行重新计算。
  2. 方法没有缓存，每次调用方法都会导致重新执行。
  3. 计算属性的 getter 和 setter 参数固定，getter 没有参数，setter 只有一个参数，而方法的参数不限。
  4. 由于有以上的这些区别，因此计算属性通常是根据已有数据得到其他数据，并在得到数据的过程中不建议使用异步、当前时间，随机数等副作用操作。
  5. 实际上，他们最重要的区别是含义上的区别，计算属性含义上也是一个数据，可以读取也可以赋值，方法含义上是一个操作，用于处理一些事情。

  • 不会改变原始数组，但总是返回一个新数组

6- 兄弟组件之间的传递(△)

  • 兄弟之间传递数据需要借助于事件中心，通过事件中心传递数据
    • 提供事件中心 ==var hub = new Vue()==
  • 传递数据方，通过一个事件触发==hub.$emit(方法名，传递的数据)==
  • 接收数据方，通过 mounted(){} 钩子中 触发==hub.$on()==方法名
  • 销毁事件 通过==hub.$off()==方法名销毁之后无法进行传递数据

4.3 组件插槽

  • 插槽的作用
    • 组件的最大特性就是复用性，而用好插槽能大大提高组件的可复用能力，插槽也就是 slot，是组件的一块 HTML 模板，相当于占位符，这块模板显示不显示，以及怎样显示由父组件来决定
    • 父组件向子组件传递内容(内容分发)
  • 插槽的使用
    • 提供一个插槽的位置
    • 插槽的内容

1- 匿名插槽

  • 注意点：
    • 父组件标签中有嵌套内容，则会将内容传递给子组件插槽，如果子组件插槽有内容，则会覆盖插槽中的内容，否则就用父组件传递过来的内容
    • 父组件标签中没有嵌套内容，不传值的话，默认使用子组件插槽中的内容

2- 具名插槽

  • 使用 <slot> 中的 “name” 属性绑定元素 指定当前插槽的名字
  • 通过 slot 属性来指定, 这个 slot 的值必须和下面 slot 组件得 name 值对应上，如果没有匹配到则放到匿名的插槽中
  • 父组件中 template 的作用：临时包裹一下数据，匹配并填充。
    • 使用场景：同时把多条文本填充到插槽中
    • 注意：属性值要写在 template 上

3- 作用域插槽

  • 作用域插槽使用场景
    • 父组件想要使用子组件的数据并进行加工处理
    • 既可以复用子组件的 slot，又可以使 slot 中模板内容不一致
  • 作用域插槽的使用
    • 定义作用域插槽：在子组件中，使用==slot==定义插槽的时候，可以通过==属性传值==的形式，为插槽传递数据
    • 使用作用域插槽：在父组件中，通过==v-slot==指令，接收并使用插槽数据

八. 路由

8.1 路由的基本概念与原理

1- 后短路由

  • 概念：根据不同的==用户 URL 请求==，返回不同的内容
  • 本质：==URL 请求地址==与==服务器资源==之间的对应关系

2- 前端路由

  • 概念：根据不同的==用户事件==，返回不同的内容
  • 本质：==用户事件==与==事件处理函数==之间的对应关系

3- SPA

  • SPA（Single Page Application）单页面应用程序：整个网站只有一个页面，内容的变化通过 Ajax 局部更新实现、同时支持浏览器地址栏的前进和后退操作

  • SPA 实现原理之一：基于 URL 地址的 hash（hash 的变化会导致浏览器记录访问历史的变化、但是 hash 的变化不会触发新的 URL 请求）

4- 简易的前端路由

  • 基于 URL 中的 hash 实现（点击菜单的时候改变 URL 的 hash，根据 hash 的变化控制组件的切换）

5- Vue Router

Vue.js 官方的路由管理器

功能：

  • 支持 HTML5 历史模式或 hash 模式
  • 支持嵌套路由
  • 支持路由参数
  • 支持编程式路由
  • 支持命名路由
