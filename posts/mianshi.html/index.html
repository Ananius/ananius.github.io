面试总结
1、h5、c3的新特性

  • H5新增
    1. 语义化标签(header, nav, article, section, aside, footer),音频视频标签(audio, video)
    2. 新增了表单类型(number, tel, serach等)–新增的表单属性(placeholder, multiple, autofocus)
  • C3新增
    1. 属性选择器和结构伪类选择器，伪元素选择器
    2. 盒模型， box-sizing: border-box (宽=宽，padding， border)
    3. 过渡， 动画 @keyframes
    4. flex 布局

    6. 文字阴影， 圆角边框，背景尺寸设置
    7. 图片边框

2、存储数据的方法

  • 三种 localstorage sessionstorage cookie
  • cookie 是一段不超过4KB 的小型文本数据，由一个名称和一个值组成，具有安全性问题，可以设置存储周期
  • localStorage: 持久存储，只要用户不主动删除就会一直存在
  • sessionStorage：面向 session 的浏览器存储，因此只存在于一个页面的生命周期 内，关闭即清除两者均采用键值对的形式存储数据

3、如何使一个盒子水平垂直居中

  • 绝对定位，left 50% top50%, 平移或者margin 50%
  • flex布局， justify-conent: center, align-item: center
  • display:table-cell
  • 利用 transform

4、如何让img图片和文字垂直居中

  • 设置行号等于高度，图片设置 vertical-align-middle

5、css盒模型

19、什么是内存泄漏？什么时候会内存泄漏呢？解决方法

  • 已经不需要的内存未及时释放或无法释放
  • 过多的使用全局变量， 当一个函数执行完之后，这个作用域中的局部变量就应该被释放
  • 通过垃圾回收机制，

20、JS 里垃圾回收机制是什么，常用的是哪种，怎么处理的？

  • 是用来防止内存泄漏的，常用标记清除 和引用计数

  • 标记清除：变量进入执行环境(函数声明变量时)，垃圾回收器会标记为进入环境， 变量离开环境，也就是(函数执行结束)，将其标记为离开环境，离开环境之后还有的变量就需要被删除。

  • 除去环境中的变量和被这些变量引用的变量，还有标记的变量就是要回收的变量

21、数组常用的方法？哪些方法会改变原数组，哪些不会

  • 会改变原数组：
    • pop (删除数组的最后一个元素并返回删除的元素)
    • push(向数组的末尾添加一个或更多元素，并返回新的长度)
    • shift(删除并返回数组的第一个元素）
    • unshift(向数组的开头添加一个或更多元素，并返回新的长度)
    • reverse(反转数组的元素顺序)
    • sort(对数组的元素进行排序)
    • splice(用于插入、删除或替换数组的元素)
  • 会改变原数组：
    • concat—连接两个或更多的数组，并返回结果。
    • every—检测数组元素的每个元素是否都符合条件。
    • some—检测数组元素中是否有元素符合指定条件。
    • filter—检测数组元素，并返回符合条件所有元素的数组。
    • indexOf—搜索数组中的元素，并返回它所在的位置。
    • join—把数组的所有元素放入一个字符串。
    • toString—把数组转换为字符串，并返回结果。
    • lastIndexOf—返回一个指定的字符串值最后出现的位置，在一个字符串中的指定位置从后向前搜索。
    • map—通过指定函数处理数组的每个元素，并返回处理后的数组。
    • slice—选取数组的的一部分，并返回一个新数组。
    • valueOf—返回数组对象的原始值

35、Object.defineProperty 和 Proxy 的区别

  • Proxy 可以直接监听对象而非属性；
  • Proxy 可以直接监听数组的变化；
  • Proxy 有多达 13 种拦截方法,不限于 apply、ownKeys、deleteProperty、has 等等 是 Object.defineProperty 不具备的
  • Proxy 返回的是一个新对象,我们可以只操作新的对象达到目的,而 Object.defineProperty 只能遍历对象属性直接修改
  • Proxy 作为新标准将受到浏览器厂商重点持续的性能优化，也就是传说中的新标准 的性能红利
  • Object.defineProperty 兼容性好，支持 IE9，而 Proxy 的存在浏览器兼容性问题, 而且无法用 polyfill 磨平，因此 Vue 的作者才声明需要等到下个大版本( 3.0 )才能用 Proxy 重 写

51、Vue 的路由实现模式：hash 模式和 history 模式

  • hash 模式的url 永远带着 # 号，在开发中默认使用这个模式，
  • 如果用户考虑url的规范的话就要使用 history模式，因为这个模式没有#号，适合做推广宣传。
  • 功能也有区别， 比如在开发app的时候有分享页面，这个分享出去的页面就是用vue或者 react做的，把这个页面分享到第三方app里，有的app是不允许出现#号的，那么就要使用history模式去掉#号
  • 但是使用history时在访问二级页面时，做刷新操作会出现 404 错误，这时候需要和后端人员配合让他配置以下 apache 或者 nginx 的url重定向就可以了

​ 首先深复制和浅复制只针对像 Object, Array 这样的复杂对象的。简单来说，浅复制只复制一层对象的属性，而深复制则递归复制了所 有层级。

​ 浅拷贝 拷贝 栈

​ 深拷贝 拷贝 堆

63 什么是闭包，优缺点，使用场景

​ 简单来说 一个函数就是一个闭包，

​ 使用场景 ：vue 页面中 data() {}

​ 优点：避免全局污染，缺点: 造成内存泄漏

64 document.write 和 innerHtml区别

​ document.write 是直接写入到页面的内容流， 每次写完关闭之后会重新调用该函数，会导致页面被重写

​ innerHTML 将页面写入某个DOM节点，不会导致页面全部重绘

65 let var const 区别

​ var 声明变量可以重复声明，而let 不可以重复声明

​ var 是不受限于块级的， 而let 是受限于块级

​ const 声明之后必须赋值，否则会报错

66 $router 和 $route 的区别

​ route 是路由信息对象，包括path、params、hash 、query、fullPath、matched、name等路由信息对象

​ router 为VueRouter 的实例。相当于一个全局的路由对象

67 路由传参的方式及路由跳转的方式

​ 传参方式 ： query (页面刷新会保留) 、params(页面刷新不会保留)

​ 跳转方式： back、push、go、replace、 router-link (声明式)

68 vue key值作用

​ 唯一值，更好的渲染 Dom

69 为什么避免 v-if 和 v-for 同时使用，v-if 和 v-for优先级

​ v-for的优先级 比 v-if 高

​ 通过 v-if 移动到容器元素，不会再重复遍历列表的每个值，取而代之的是，我们检查它一次，且不会在v-if为否的时候运算 v-for
